# MicroBounty — Frontend

**Live:** [micro-bounty.vercel.app](https://micro-bounty.vercel.app)

## Stack

| | |
|---|---|
| Framework | Next.js 15 (App Router) + TypeScript |
| Styling | Tailwind CSS |
| Wallet | Reown AppKit (MetaMask, SubWallet, Talisman) |
| Blockchain | ethers.js v6 |
| Notifications | Sonner |

## Prerequisites

- Node.js v18+
- Deployed MicroBounty contract (see `../contract/README.md`) or use the existing testnet deployment
- [Reown](https://reown.com) project ID (free)

## Setup
```bash
cd frontend
npm install
cp .env.example .env.local
```

`.env.local`:
```env
NEXT_PUBLIC_REOWN_PROJECT_ID=your_reown_project_id
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

`lib/abis/contract-addresses.json` is generated by the contract deploy script. For the live testnet contract it is already present in the repo.

## Run
```bash
npm run dev        # Development
npm run build      # Production build
npm run start      # Serve production build
```

## Deploy to Vercel
```bash
npm install -g vercel
vercel
```

Add `NEXT_PUBLIC_REOWN_PROJECT_ID` and `NEXT_PUBLIC_APP_URL` in the Vercel dashboard.

## Key Design Decisions

### DOT has 10 decimals, not 18

Never use `ethers.formatEther` for DOT amounts. Use:
```ts
ethers.formatUnits(amount, 10)   // display
ethers.parseUnits(input, 10)     // input → raw
```

This is applied consistently in `WalletContext` and `BountyContext`.

### ERC20 approval flow

Creating a stablecoin bounty requires two wallet confirmations:
1. `approve()` — authorise the contract to spend tokens
2. `createBounty()` — lock tokens in escrow

`CreateBountyForm` handles both steps with clear per-step UI feedback.

### RPC propagation delay

After `approveBounty()` or `cancelBounty()`, the testnet RPC node can take a few seconds to reflect updated `totalValueLocked` in view calls. `BountyContext` fetches stats immediately after confirmation and again after a 3-second delay to keep the UI accurate.
